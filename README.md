# 古风叙事冒险文字游戏 — 设计计划书

## 一、项目概览

| 维度 | 决策 |
|------|------|
| **游戏名称** | 待定（建议方向：归园田居 / 山水谣 / 桃源纪事） |
| **类型** | 古风叙事冒险 + 轻经营 |
| **平台** | Web 网页（移动端 + 桌面端响应式） |
| **技术栈** | Vue 3 + TypeScript + Vite + TailwindCSS + Pinia |
| **规模** | MVP 快速原型，1-2 周可玩版本 |
| **参考项目** | 桃源乡 (github.com/setube/taoyuan) |

---

## 二、游戏概念

### 故事梗概

> 你原是长安城中一个落魄书生，屡试不第，心灰意冷。一日收到远方来信——素未谋面的叔父在终南山下留给你一间草庐和几亩薄田。你决定离开喧嚣，去往那个叫「桃源」的地方，重新开始。

### 核心循环

```
探索剧情（文字选择） → 获得资源/解锁NPC → 经营田地 → 触发新剧情 → 循环
```

### 不同之处

与桃源乡的区别——我们的重点是**叙事驱动**而非纯经营：

| | 桃源乡（参考） | 本项目 |
|---|---|---|
| 驱动方式 | 经营驱动 | 故事驱动 |
| 操作密度 | 高，每季大量农活 | 中低，关键选择决定走向 |
| 系统数量 | 14 个系统 | MVP 4 个核心系统 |
| NPC 数量 | 34 位 | 3-5 位深度角色 |
| 战斗 | 矿洞回合制 | 暂不涉及 |

## 三、MVP 核心系统

### 系统 1：故事引擎

```
结构：
  序章（到来）
    ├── 第1章：扎根（认识村民，学会耕种）
    ├── 第2章：邻里（解决村庄矛盾，建立关系）
    ├── 第3章：传承（发现村庄秘密，做出关键抉择）
    └── 尾章：结局（根据选择分支，3种结局）

每章包含：
  - 5-8 个故事节点
  - 每个节点 2-3 个选择分支
  - 选择影响：好感度 / 资源 / 后续剧情走向
```

### 系统 2：经营系统（简化版）

| 要素 | MVP 范围 |
|------|----------|
| 田地 | 初始 4 块，可扩至 6 块 |
| 作物 | 4 种（小麦/白菜/萝卜/茶树），按季节区分 |
| 季节 | 春/夏/秋/冬 循环 |
| 体力 | 每日 100 点，农活消耗 |
| 货币 | 铜钱，售卖作物获得 |
| 售卖 | 简单的 NPC 商店 |

### 系统 3：NPC 与好感度

MVP 3 位 NPC：

| NPC | 身份 | 功能 | 喜好 |
|-----|------|------|------|
| 药姑·云娘 | 村口草药铺主人 | 购买种子、恢复体力 | 草药、茶叶 |
| 樵夫·老陈 | 后山老樵夫 | 传授耕种技巧、出售工具 | 白酒、肉食 |
| 少女·小荷 | 邻家女孩 | 剧情主线关键人物 | 鲜花、糕点 |

好感度 0-100，每到 25/50/75/100 触发特殊事件。

### 系统 4：时间与天气

- 每天分为 早晨/午后/傍晚/夜间 四个时段
- 天气：晴/雨/雪（冬）三种
- 每个操作消耗一个时段
- 季节轮替（每季 10 天，MVP 先从春开始）

## 四、UI 界面规划

```
┌──────────────────────────────────┐
│  状态栏：季节 · 天数 · 铜钱 · 体力  │
├──────────┬───────────────────────┤
│          │                       │
│  场景    │   故事文本区域         │
│  插图    │   对话 / 叙述 / 事件   │
│  (CSS画) │                       │
│          │   [选项A]             │
│  或      │   [选项B]             │
│  像素画  │   [选项C]             │
│          │                       │
│          ├───────────────────────┤
│          │  快捷面板              │
│          │  [田地] [背包] [社交]  │
│          │  [商店] [存档] [设置]  │
└──────────┴───────────────────────┘
```

移动端：纵向布局，场景图在上，文本和选项在下。

### 配色方案

基于古风中国色：
- 背景：宣纸色 `#f5f0e8`
- 主文字：墨色 `#2c2416`
- 强调色：朱砂 `#c34043`
- 辅助色：竹青 `#5a9e6f`
- 金色点缀：赤金 `#c8a45c`
- 字体：使用思源宋体 / 站酷小薇

## 六、开发计划（MVP）

| 阶段 | 内容 | 预计时间 |
|------|------|----------|
| **第1步：项目搭建** | Vite + Vue3 + TS + Tailwind 脚手架，路由和主题样式 | Day 1 |
| **第2步：核心状态** | Pinia stores：game、farm、story、npc，类型定义 | Day 1-2 |
| **第3步：故事引擎** | 剧情数据格式设计，编写序章 + 第1章内容 | Day 2-3 |
| **第4步：游戏主界面** | StoryPanel + StatusBar + 场景展示 | Day 3-4 |
| **第5步：经营系统** | 田地管理、种植/收获、季节轮替 | Day 4-5 |
| **第6步：NPC 系统** | 对话、送礼、好感度、事件触发 | Day 5-6 |
| **第7步：存档系统** | localStorage 存档/读档，3 个槽位 | Day 6-7 |
| **第8步：内容补齐** | 第2-3章剧情、结局分支 | Day 7-8 |
| **第9步：打磨** | 过渡动画、音效（可选）、移动端适配 | Day 8-10 |
